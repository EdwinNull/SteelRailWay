## 0. 论文核心思想速览

TRD 在多模态工业异常检测（RGB + 3D）上，把经典的 **Reverse Distillation (RD)** 框架扩展为 **双分支 Tuned Reverse Distillation**：

1. **Reverse Distillation 框架**：Teacher 是冻结的 ImageNet 预训练 Wide-ResNet50，Student 是一个“反向”的 ResNet（layer 顺序与通道数倒过来）。Student 只在正常样本上学习去 **重建** Teacher 的多尺度特征。测试时用 Teacher/Student 特征的余弦距离当作异常分数。
2. **双 Teacher + 双 Student（RGB 与 3D 两路独立 RD）**：每个模态都有自己的 Teacher/Student，分别产生各自的异常图。
3. **Crossmodal Tuner（跨模态调谐器，本文最关键创新）**：训练时，把 **另一个模态的 Teacher 特征** 经过两条不同的 “Tuner” 通路投影到当前模态 Teacher 的特征空间，然后注入到当前模态 Student 的解码路径中：
    * **Bottleneck Tuner**（对应 `NormalProjector_8`）：先下采样到全局瓶颈再上采样回原分辨率，学习“全局互补线索”。
    * **Per-scale Tuner**（对应 `Projector_Amply_upsample`）：在每个尺度上做通道扩张-压缩，学习“局部互补细节”。
    并用 **可学习的 softmax 权重** 把 Tuner 输出与 Student 主干的反卷积特征融合（对应 `ResNet_Skip.forward_fuse`）。
4. **多路蒸馏监督**：4 路 Student 输出（Tuner 各自输出 + Student 主干 + 主干与 Tuner 融合版本）都被拉向 Teacher，形成深度的跨模态正则。
5. **3D 模态表示**：把原始 XYZ 点云转成 **表面法线图 (surface normal map)** 作为 3D 输入（RGBN 版），比直接用深度图更能捕捉表面方向的异常；另外也提供 RGB-D 版本作对照。

## 2. Teacher 与 Student 的“反向 ResNet”

### 2.1 Teacher：[encoders.py](models/encoders.py)

```python
class ResNet50Encoder(torch.nn.Module):
    def __init__(self, pretrained=True):
        self.encoder = wide_resnet50_2(pretrained=pretrained)
    def forward(self, x):
        return self.encoder(x)   # -> [feature_a, feature_b, feature_c]
```

调用 [resnet.py:233-246](models/resnet.py#L233-L246)：依次经过 `conv1→bn1→relu→maxpool→layer1→layer2→layer3`，输出三个尺度（**注意：只用到 layer1/2/3，丢弃 layer4**）：

| 索引 | 对应层 | 通道数 | 分辨率（输入 256） |
| --- | --- | --- | --- |
| `[0]` feature_a | layer1 | 256 | 64×64 |
| `[1]` feature_b | layer2 | 512 | 32×32 |
| `[2]` feature_c | layer3 | 1024 | 16×16 |

训练中 Teacher 的参数被冻结（[train_MVTec3D_rgbn.py:270](train_MVTec3D_rgbn.py#L270)），只做前向。

### 2.2 反向 ResNet（de_resnet）：[de_resnet.py](models/de_resnet.py)

普通 ResNet 是“通道由少到多、分辨率由大到小”；反向 ResNet 把两者都颠倒：`inplanes` 初始为 `512*4=2048`，`layer1/2/3` 的 `planes` 分别是 256/128/64，对应输出通道是 1024/512/256。每层的下采样 `Conv` 被替换为反卷积 `deconv2x2`（[de_resnet.py:68-71](models/de_resnet.py#L68-L71)、[de_resnet.py:125-128](models/de_resnet.py#L125-L128)），从而实现“上采样 + 通道减少”的逆过程。输出顺序为 `[feature_c, feature_b, feature_a]`（深→浅），通道依次 1024/512/256 —— **与 Teacher 的 `[a, b, c]`（浅→深）一一对应**，后续蒸馏按位置索引做逐尺度 loss。

### 2.3 带跨模态注入的 Student：`ResNet_Skip.forward_fuse`

[de_resnet.py:378-544](models/de_resnet.py#L378-L544) 在反向 ResNet 里加入了**每一层一组可学习权重** `w_a, w_b, w_c ∈ R²`，并通过 softmax 融合“自身反卷积输出”与“跨模态 tuner 输出”：

```python
feature_a_0 = self.layer1(x)              # 自身反卷积输出（1024ch）
w1, w2 = softmax(self.w_a)                # 可学习融合权重
feature_a = w1 * feature_a_0 + w2 * x_assit[2]   # 与深层 tuner 对齐
feature_b_0 = self.layer2(feature_a)
feature_b   = w1' * feature_b_0 + w2' * x_assit[1]
feature_c_0 = self.layer3(feature_b)
feature_c   = w1'' * feature_c_0 + w2'' * x_assit[0]
return [feature_c, feature_b, feature_a], [feature_c_0, feature_b_0, feature_a_0]
```

**关键**：返回两组特征：
- 第一组是**融合后**（"skip-fused"）；
- 第二组是**融合前**（纯 student 主干）。

两组都会在训练阶段分别被蒸馏到 Teacher（见 §4），这就是论文中 "tuner 只影响正常样本分布，但通过多路监督让 Student 主干也学到足够强的重建能力" 的实现。

## 3. Crossmodal Tuner（跨模态调谐器）

[projector.py](models/projector.py) 定义了 **两条 tuner**。它们都接收“**另一模态 Teacher 的 3 尺度特征**”作为输入，并把它们投影到“**当前模态 Teacher 的特征空间**”。

### 3.1 `NormalProjector_8`（Bottleneck Tuner） [projector.py:101-161](models/projector.py#L101-L161)

> 名字里的 "_8" 指在 `conv_a` 分支把空间分辨率最多下采 8 倍（64→32→16→8）。**这是一个 "编码-解码" 的瓶颈结构**，而不是文档旧版中描述的“3 次 stride=2 下采再上采”那样简单。

- `conv_a`（输入 256ch/64×64，对齐 teacher_a）：
  3 级 stride-2 卷积把通道升到 `256→512→1024→2048` 并把分辨率压到 `64→32→16→8`；
  3 级反卷积再把通道降到 `2048→1024→512→256` 并把分辨率还原到 64×64。
- `conv_b`（输入 512ch/32×32，对齐 teacher_b）：
  2 级 stride-2 卷积 `512→1024→2048`、压到 8×8；2 级反卷积回 `2048→1024→512`、回到 32×32。
- `conv_c`（输入 1024ch/16×16，对齐 teacher_c）：
  1 级 stride-2 卷积 `1024→2048`，1 级反卷积 `2048→1024`，回到 16×16。

> **论文动机**：每个尺度都走到同一个“全局 8×8”瓶颈再解码，等价于让跨模态特征先被压到只剩“语义级全局上下文”，再上采重建。这对“正常样本分布建模”非常重要——异常纹理会在瓶颈处被过滤掉。

`forward` 返回 `[proj_a, proj_b, proj_c], [proj_a, proj_b, proj_c]`，两份同一内容；解码器用其中一份（`proj_filter`）送入 **bottleneck 融合层**（见 §3.3）。

### 3.2 `Projector_Amply_upsample`（Per-scale Tuner） [projector.py:48-98](models/projector.py#L48-L98)

每个尺度各自走一个 “3×3 保形 → 1×1 升通道至 2× → 1×1 降通道回原通道”的小瓶颈（通道上的 "amply & compress"）：

| 分支 | 通道路径 | 分辨率 |
| --- | --- | --- |
| `conv_a` | 256 → 256 → 512 → 256 | 64×64 保持 |
| `conv_b` | 512 → 512 → 1024 → 512 | 32×32 保持 |
| `conv_c` | 1024 → 1024 → 2048 → 1024 | 16×16 保持 |

**论文动机**：与 `NormalProjector_8` 形成互补——这条 tuner 只在“通道维度”扩张再压缩，完全不做空间降采样，保留局部细节。最后在 `ResNet_Skip` 的每一层与 student 特征做 softmax 加权融合（§2.3）。

### 3.3 Bottleneck 融合：`bn_fuse` / `BN_fuse_layer` [resnet.py:447-522](models/resnet.py#L447-L522)

在 Student 解码的**起点**（瓶颈），把主模态 teacher 的 3 尺度 `x` 与跨模态 tuner 输出 `x_assist`（`proj_filter`）**按尺度 concat 再做卷积**：

```python
# l1: 把最浅的两层 concat 后做两次 stride-2 conv，对齐到最深分辨率
l1 = relu(bn2(conv2( relu(bn1(conv1( cat([x[0], x_assist[0]],1) )))) ))
l2 = relu(bn3(conv3( cat([x[1], x_assist[1]], 1) )))
l3 = relu(bn4(conv4( cat([x[2], x_assist[2]], 1) )))
feature = cat([l1, l2, l3], 1)          # 最终瓶颈张量
output  = self.bn_layer(feature)        # 进入 ResNet_Skip.layer1 的输入
```

这是**跨模态信息进入 Student 的第一道门**；随后解码每一层还会再通过 `Projector_Amply_upsample` + softmax 权重注入一次（§2.3）。

### 3.4 双模态解码器整体：[decoders.py](models/decoders.py)

```python
class ResNet50DualModalDecoder(nn.Module):
    def __init__(self):
        self.bn               = bn_fuse()                        # BN_fuse_layer（§3.3）
        self.decoder          = de_wide_resnet50_2_skip()        # ResNet_Skip（§2.3）
        self.projector_filter = NormalProjector_8()              # Bottleneck Tuner（§3.1）
        self.projector_amply  = Projector_Amply_upsample()       # Per-scale Tuner（§3.2）

    def forward(self, x, x_assist):
        proj_filter, x_assist_proj = self.projector_filter(x_assist)  # 跨模态 -> 当前模态空间
        proj_amply                = self.projector_amply(x_assist)
        x_bn                      = self.bn(x, proj_filter)            # 瓶颈融合
        x_amply, x                = self.decoder.forward_fuse(x_bn, proj_amply)
        return proj_filter, proj_amply, x, x_amply
```

**四个返回值**（顺序很关键，下面蒸馏 loss 完全依赖这个顺序）：

| 位置 | 变量名（train 里） | 含义 | 用于蒸馏的目标 |
| --- | --- | --- | --- |
| 0 | `proj_d`       | `proj_filter`，Bottleneck-Tuner 输出（已对齐主模态空间） | 主模态 Teacher |
| 1 | `proj_d_amply` | `proj_amply`，Per-scale-Tuner 输出                      | 主模态 Teacher |
| 2 | `output_Sr`    | `x`，`ResNet_Skip` **融合前** 的纯 student 特征        | 主模态 Teacher |
| 3 | `output_Sr_am` | `x_amply`，**融合后** 的 student 特征                   | 主模态 Teacher |

> 旧版文档把 2 和 3 的含义搞反了：**位置 2 是 "_0" 分支（未融合）**，**位置 3 才是融合后的 amplified 版本**。测试阶段只使用位置 3（融合后）生成异常图。

## 4. 训练脚本：[train_MVTec3D_rgbn.py](train_MVTec3D_rgbn.py)

### 4.1 主循环逻辑

每个类别独立训练 200 epoch；每个 epoch 做如下事情（[train_one_epoch](train_MVTec3D_rgbn.py#L71)）：

```python
with torch.no_grad():
    output_Tr = teacher_rgb(rgb_image)       # RGB Teacher 多尺度特征
    output_Td = teacher_depth(normal_image)  # Normal Teacher 多尺度特征

# --- RGB 分支：以 RGB 为主模态，Normal 做跨模态调谐器输入 ---
proj_d, proj_d_amply, output_Sr, output_Sr_am = student_rgb(output_Tr, output_Td)
loss_rgb =   loss_distil(output_Sr,    output_Tr) \
           + loss_distil(proj_d,       output_Tr) \
           + loss_distil(proj_d_amply, output_Tr) \
           + loss_distil(output_Sr_am, output_Tr)

# --- Normal 分支：以 Normal 为主模态，RGB 做跨模态调谐器输入 ---
proj_r, proj_r_amply, output_Sd, output_Sd_am = student_depth(output_Td, output_Tr)
loss_depth = loss_distil(output_Sd,    output_Td) \
           + loss_distil(proj_r,       output_Td) \
           + loss_distil(proj_r_amply, output_Td) \
           + loss_distil(output_Sd_am, output_Td)
```

- **对称的双向蒸馏**：RGB-Student 用 Normal 特征做辅助去重建 RGB Teacher；Normal-Student 用 RGB 特征做辅助去重建 Normal Teacher。
- **4 路监督**：每路 student 都有 4 个输出都被拉向 Teacher。其中把 `proj_d` / `proj_d_amply` 直接蒸馏到 Teacher 的意义是——强迫 **Tuner 本身** 输出就要落在主模态 Teacher 的分布里，而不仅仅是“辅助项”。
- **双优化器** `optimizer_rgb` / `optimizer_depth` 分别独立反传更新，互不干扰。
- **AMP 支持**（[train_MVTec3D_rgbn.py:117-152](train_MVTec3D_rgbn.py#L117-L152)）：CUDA 下启用 `torch.cuda.amp.GradScaler` + `autocast(fp16)` 提速省显存。

### 4.3 Validation 统计 → Test Z-score 标准化

`valid()`（[train_MVTec3D_rgbn.py:483-525](train_MVTec3D_rgbn.py#L483-L525)）在验证集上累积 RGB/Normal 两条分支的异常图，计算每条分支的全局 `(mean, std)`。测试阶段用它们做 **Z-score 标准化**，再相加融合——这一步大幅减小两个模态的尺度差异（否则量纲不一样直接相加会被其中一个压过）：

```python
amap_rgb   = (amap_rgb   - mean_r) / std_r
amap_depth = (amap_depth - mean_d) / std_d
amap       = amap_rgb + amap_depth
amap       = gaussian_filter(amap, sigma=4)   # 平滑
```

```python
for i in range(len(fs_list)):
```

## 6. 数据集：RGB-Normal 预处理（[dataset_rgbn.py](dataset/dataset_rgbn.py)）

### 6.1 从 XYZ tiff → 前景 mask → 深度归一化 → 空洞填充 → 法线图

[get_normal_image](dataset/dataset_rgbn.py#L117-L166)：

1. 读取 `xyz/*.tiff`，取第 3 通道作为原始深度。
2. `get_plane_mask`（[geo_utils.py](dataset/geo_utils.py)）：从图像四角采点拟合背景平面，距离大于阈值者视为前景。
3. `fill_plane_mask`：用形态学闭运算补洞。
4. 只保留前景像素的深度，并在前景内做 min-max 归一化。
5. `fill_depth_map`：用 `unfold/fold` 的局部均值迭代填补零值像素。
6. **`depth_to_normal_map`**（[dataset_rgbn.py:22-35](dataset/dataset_rgbn.py#L22-L35)）：
   ```python
   dzdx = cv2.Sobel(depth, CV_64F, 1, 0, ksize=k)
   dzdy = cv2.Sobel(depth, CV_64F, 0, 1, ksize=k)
   normal = [-dzdx, -dzdy, 1]; normal /= ||normal||
   normal_img = (normal + 1) / 2                  # 映射到 [0,1]
   ```
7. 乘回前景 mask，`cv2.resize` 到 `img_size`，输出 `3×H×W` tensor。

> Sobel 核大小 `k` 控制法线平滑度；对表面起伏小、材质平整的类别（cookie）取较小 k，对粗糙纹理（dowel/foam/tire）取较大 k。

### 6.2 归一化缓存

`MVTecADRGBNDataset` 默认对 **训练集** 缓存 `depth_path → (normal_tensor, mask_tensor)`（[dataset_rgbn.py:73-78](dataset/dataset_rgbn.py#L73-L78)），避免每个 epoch 重复做“读 tiff + 平面拟合 + Sobel”的昂贵操作。

### 6.3 全局深度范围

训练集会遍历所有样本，计算**全局** `global_min, global_max`（取每张图的 min/max 再扩大到 `[0.9×min, 1.1×max]`，[dataset_rgbn.py:50-57](dataset/dataset_rgbn.py#L50-L57)），为**测试阶段**保持一致的深度缩放提供锚点。

## 7. 损失函数：[losses.py](utils/losses.py)

### 7.1 主蒸馏损失：余弦相似度（[losses.py:21-42](utils/losses.py#L21-L42)）

```python
def loss_distil(feature_s, feature_t):
    cos = torch.nn.CosineSimilarity()
    loss = 0.0
    for i in range(len(feature_s)):
        s = feature_s[i].view(B, -1)
        t = feature_t[i].view(B, -1)
        loss += torch.mean(1 - cos(s, t))
    return loss
```

> 注意：这里是把整张 `C×H×W` 展平成一维向量后在 batch 维算余弦相似度，得到的是 **每张图全局方向的一致性**；像素级的方向一致性在 `cal_anomaly_map` 测试时才逐点计算。这一设计让**训练目标偏向全局分布对齐**，而测试时用**逐像素偏差**当异常分数，两者互补。

### 7.2 备选：L2 距离 `loss_l2`、像素级 `loss_distil_pixel`

后者会对每两个像素间计算相似度矩阵，并对比 student/teacher 的相似度结构，更精细但计算量极大；本仓库主训练循环只使用 `loss_distil`。

## 10. 训练数据流（全流程总览）

```
┌────────────────────────────────────────────────────────────────────────┐
│                       TRD 训练一个 step 的数据流                       │
└────────────────────────────────────────────────────────────────────────┘
     RGB image                          Normal image (3 通道)
         │                                      │
     Teacher_RGB  (frozen)               Teacher_N  (frozen)
     [a_r, b_r, c_r]                     [a_n, b_n, c_n]
         │                  \   /                │
         │                   \ /                 │
         │            ┌───── X ─────┐            │
         │            │             │            │
         ▼            ▼             ▼            ▼
 ┌─────────── Student_RGB ───────────┐  ┌─────────── Student_N ───────────┐
 │ proj_filter = NormalProjector_8(n)│  │ proj_filter = NormalProjector_8(r)│
 │ proj_amply  = Projector_Amply(n)  │  │ proj_amply  = Projector_Amply(r)  │
 │ x_bn        = BN_fuse(r, proj_f)  │  │ x_bn        = BN_fuse(n, proj_f)  │
 │ (x_amply,x) = ResNet_Skip(x_bn,   │  │ (x_amply,x) = ResNet_Skip(x_bn,   │
 │               proj_amply, softmax)│  │               proj_amply, softmax)│
 │ → proj_f, proj_a, x, x_amply      │  │ → proj_f, proj_a, x, x_amply      │
 └──────────┬────────────────────────┘  └──────────┬────────────────────────┘
            │ 4 路全部蒸馏 → Teacher_RGB            │ 4 路全部蒸馏 → Teacher_N
            ▼                                        ▼
         loss_rgb                                 loss_depth
     optimizer_rgb.step()                    optimizer_depth.step()
```

测试时只走实线部分，把两条分支的 `x_amply` 分别与对应 Teacher 算余弦距离得到异常图，做 Z-score 标准化后相加并高斯平滑。

| 论文中的部件 | 本仓库实现 |
| --- | --- |
| Frozen encoder $E(\cdot)$ | `ResNet50Encoder` / `wide_resnet50_2` |
| Reverse decoder $D(\cdot)$ | `de_wide_resnet50_2_skip` + `ResNet_Skip.forward_fuse` |

## 12. FAQ（工程层面）

**Q1：为什么 Teacher 只取 layer1~3 不取 layer4？**
A：layer4 分辨率太低（8×8）对定位像素级异常不利；三尺度已经覆盖 local–semantic。此外反向解码器也只有 3 层，保持对称。

**Q2：为什么 `loss_distil` 把 H×W 展平算余弦？**
A：训练阶段只需要 student 的全局特征分布与 teacher 一致；逐像素对齐在测试时用 `cal_anomaly_map` 实现即可。展平版可微且梯度稳定。

**Q3：`NormalProjector_8` 和 `Projector_Amply_upsample` 能不能只留一个？**
A：论文消融表显示两者互补——前者提供“全局上下文补全”，后者提供“通道级细节放大”；去掉任一 AUPRO 都明显下降。

**Q4：softmax 融合权重会不会退化成全走 x_assit？**
A：不会，因为 `proj_d` / `proj_d_amply` / `output_Sr` / `output_Sr_am` **都**要蒸馏到 Teacher；如果 softmax 把 `feature_a_0` 这一路压成 0，`output_Sr`（未融合）将无法重建 Teacher，loss 会反向拉回平衡。

**Q5：Windows 下 `num_workers=8` 有时反而比 16 快？**
A：Windows 的 `spawn` 多进程开销远高于 Linux `fork`，再加上 Sobel/tiff 读取已被 `cache_normals` 缓存，过多 worker 反而被 IPC 开销拖累。作者的经验注释在 [train_MVTec3D_rgbn.py:531](train_MVTec3D_rgbn.py#L531)。

## 14. 本文档相对旧版的修正

| 旧版描述 | 实际代码 |
| --- | --- |
| “`NormalProjector_8` 由 3 次 stride=2 下采 + 3 次反卷积” | 实际 **每个分支各自不同**：conv_a 下采 3 次至 /8 再上采；conv_b 下采 2 次；conv_c 只下采 1 次。目标是三尺度统一压到 8×8 的全局瓶颈再分别还原 |
| “`Projector_Amply_upsample` 先扩 2× 再压回原通道” | **还额外包含一个 3×3 保形卷积**，共 3 级而不是 2 级；且每个尺度的中间通道都是该尺度的 2× |
| “student 的 4 个输出中，位置 2 是融合后、位置 3 是未融合” | 正好相反：**位置 2 是 `x`（未融合，"_0" 分支）**，**位置 3 是 `x_amply`（softmax 融合后）**；测试时异常图用位置 3 |
| “跳跃连接固定相加” | 实际为 **每层独立的可学习 softmax 权重** `w_a / w_b / w_c`，这是 TRD "tuned" 一词的由来之一 |
| “`cal_anomaly_map` 有 add/mul 两种模式，测试默认 mul” | 训练脚本里 `test()` 显式传了 `amap_mode='add'`，默认实参 mul 并未被使用 |
